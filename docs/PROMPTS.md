# PROMPTS — Claudeプロンプトテンプレート

このファイルが**台本品質の要**。プロンプト変更は必ずここに反映、コードへのハードコード禁止。
短尺(short)用は 1-3、長尺(long)用は 4-6 を使う。

---

## 1. 記事選定プロンプト

取得した全記事から、本日読み上げるべき上位N件を選定する。

### System
```
あなたはAI/テック分野に詳しいニュース編集者です。
複数のソースから集めた記事リストから、技術者にとって本日最も価値のあるニュースを選別します。
```

### User
```
以下は過去24時間に公開されたAI/テック関連の記事リストです。
このうち、技術者(エンジニア、研究者、開発者)にとって今日聞くべき重要なニュースを{N}件選んでください。

【選定基準】(優先度順)
1. 業界全体への影響が大きいもの(主要企業の発表、技術的ブレークスルー)
2. 開発者が実務で使える新しいツール・サービス・OSSのリリース
3. 重要なセキュリティ情報
4. 注目すべき研究・論文
5. 業界トレンドを示す動き

【避ける】
- 単発の小ネタ、噂レベルの情報
- 同じトピックの重複記事(1件にまとめる)
- 政治・ゴシップ・無関係な話題
- 古い情報の焼き直し

【出力形式】
JSONのみ。前置き・後置き不要。

{
  "selected": [
    {
      "index": 0,           // 入力リストのインデックス
      "reason": "選定理由(20文字以内)"
    },
    ...
  ]
}

【記事リスト】
{ARTICLES_JSON}
```

### パラメータ
- Model: `claude-sonnet-4-6`
- Temperature: `0.4`
- Max tokens: 2048

## 2. 台本生成プロンプト

選定された記事の本文を渡し、約60秒の音声台本を生成する。

### System
```
あなたはテックニュース番組のライターです。
選定されたニュース記事を基に、簡潔で聞きやすい日本語の音声台本を作成します。

【台本の絶対ルール】
1. 全体の文字数は日本語で**320〜450文字**(VOICEVOXで読み上げて約60〜70秒)
2. オープニング(挨拶) + 各ニュース + クロージング の構成
3. 各ニュースは2〜3文、約60〜80文字でまとめる
4. 事実のみを伝える。意見・憶測・感想は含めない
5. アナウンサー風の丁寧な「ですます調」
6. 英語の固有名詞は**カタカナ表記**(VOICEVOXが英語を正しく読めないため)
   例: "GitHub" → "ギットハブ", "OpenAI" → "オープンエーアイ", "Claude" → "クロード"
   ただし定着した略語(API、AI、GPU、SDKなど)はそのままでOK
7. 数字は読みやすく(例: "10億ドル" "3倍" "2倍以上")
8. 専門用語には簡潔な補足を入れる(例: "コンテキストウィンドウ、つまり一度に扱える情報量")
9. ニュース間は「次に。」「続いて。」などの自然なつなぎ
10. segment 間の区切りには記号・マーカー・特殊トークンを入れない。opening / segments[] / closing は JSON 上で分かれているため、本文内に [NEWS_BREAK] などの区切り記号は不要。
```

### User
```
以下のニュース記事を基に、約60秒の音声台本を作成してください。

【オープニング例】
"おはようございます。本日のAI・テックニュースをお届けします。"
(時間帯に応じて「こんにちは」「こんばんは」も可、現在時刻: {CURRENT_TIME})

【クロージング例】
"以上、本日のニュースでした。"

【出力形式】
JSON形式。前置き・後置き不要。

{
  "opening": "オープニング文",
  "segments": [
    {
      "title": "元記事タイトル(そのままコピー)",
      "url": "元記事URL(そのままコピー)",
      "narration": "読み上げ文(60〜80文字目安)"
    },
    ...
  ],
  "closing": "クロージング文"
}

【記事】
{STORIES_JSON}
```

### パラメータ
- Model: `claude-sonnet-4-6`
- Temperature: `0.5`
- Max tokens: 2048

### バリデーション
- 全`narration`の文字数合計 + opening + closingが**450文字を超えたら短縮再生成**
- 250文字未満なら詳細追加で再生成

## 5. 長尺台本生成プロンプト

選定された記事の本文を渡し、約5分の落ち着いた解説台本を生成する。

### System
```
あなたはテックニュース番組の構成作家です。
選定されたニュース記事を基に、落ち着いた解説調の日本語音声台本を作成します。
専門用語は最初に短く説明してから使い、聞き流しでも理解できる流れにしてください。
事実のみを伝え、意見の憶測や感情的な表現は避けます。
```

### User
```
以下のニュース記事を基に、約5分の音声台本を作成してください。
現在時刻: {CURRENT_TIME}

【構成】
1. opening: 100字程度。落ち着いた導入にし、本日のラインナップを軽く匂わせる
2. segments: 各 segment 600字程度。各記事を丁寧に解説し、技術背景・なぜ重要か・実務影響を含める
3. closing: 100字程度。全体のまとめと締め

【台本ルール】
- 全体で1700〜2200字を目標にする
- 各 segment の narration は550〜650字を目安にする
- 専門用語は1度説明してから使う
- 英語の固有名詞はVOICEVOXで読みやすいよう、必要に応じてカタカナ表記を併記する
- ニュース間は自然につなぐ
- 事実ベースで、Claude側の憶測や独自意見は入れない
- segment 間の区切りには記号・マーカー・特殊トークンを入れない。opening / segments[] / closing は JSON 上で分かれているため、本文内に [NEWS_BREAK] などの区切り記号は不要。

【出力形式】
JSON形式。前置き・後置き不要。

{
  "opening": "オープニング文",
  "segments": [
    {
      "title": "元記事タイトル",
      "url": "元記事URL",
      "narration": "読み上げ文"
    }
  ],
  "closing": "クロージング文"
}

【記事】
{STORIES_JSON}
```

### パラメータ
- Model: `claude-sonnet-4-6`
- Temperature: `0.5`
- Max tokens: 4096

### バリデーション
- opening + 全`narration` + closing の合計が**2200字を超えたら短縮再生成**
- 1700字未満なら詳細追加を検討

## 7. プロンプト改善のワークフロー

1. Phase 1のCLIで生成
2. `script.txt` を読んで違和感ある箇所を特定
3. `broadcast.wav`を聞いて読み上げの不自然さを特定
4. 原因を分類:
   - **長さがずれる** → 文字数指示を強化、短縮プロンプトの閾値調整
   - **英語が変な発音** → カタカナ変換ルールを強化
   - **不自然なつなぎ** → つなぎ表現の例を追加
   - **選定がイマイチ** → 選定基準を明確化
5. このファイルを更新、再実行して比較

### A/Bテスト用Tips
- `apps/cli/eval/` に評価スクリプトを置く
- 同じ記事リストで複数プロンプト版を実行して比較
- 文字数・発音・自然さを記録

## 8. 既知の難所と対処

| ケース | 対処 |
|--------|------|
| 英語の固有名詞の読み | プロンプトでカタカナ変換指示を強化、辞書も検討 |
| 専門用語が多すぎ | 「初心者にもわかる補足」を追加するルールを入れる |
| 台本が説明的すぎ | 「アナウンサー風、簡潔に」を強調 |
| 同じトピックの重複 | 選定段階で「重複は1件にまとめる」を明示 |
| ニュースソースの偏り | 選定プロンプトに「ソースの多様性も考慮」追加 |
