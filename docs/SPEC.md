# SPEC — 機能仕様

## ターゲットユーザー

毎日のAI/テック業界の動きを**短時間でキャッチアップしたい人**。
通勤中・朝の支度中・作業の合間に「ながら聞き」する用途を想定。

## ユーザーストーリー

### US-1: 今日のニュースを聞く(MVPの中核)

> ユーザーとして、ボタンを1回押すと、その日のAI/テックニュース3〜5件を要約した約1分の音声を聞きたい。

**フロー:**
1. トップページにアクセス
2. 「今日のニュースを生成」ボタンを押す
3. 進捗表示(取得中 → 要約中 → 音声生成中)
4. 完了したら自動再生開始
5. 再生中は読み上げ中のニュース見出しと出典リンクを表示

**所要時間目安:** ボタン押下から再生開始まで30秒〜1分

### US-2: 出典を確認する

> ユーザーとして、気になったニュースは元記事を読みたい。

**仕様:**
- 再生中、現在読み上げ中のニュースに対応する記事カードがハイライト
- カードクリックで元記事を新規タブで開く
- 全ニュースのリストは画面下部に表示(再生終了後も確認可能)

### US-3: 過去の放送を聞き直す(Phase 4)

> ユーザーとして、昨日生成した放送をもう一度聞きたい。

**仕様:**
- 履歴ページで過去N日分の放送を一覧表示
- 各放送の見出しと音声ファイルを保存(MVPはlocalStorageで日付ベース管理)

## 画面要件

### `/` トップページ(MVPの主画面)
- 中央に大きい「今日のニュースを生成」ボタン
- 過去24時間以内の生成済み放送があれば「最後の放送を再生」ボタンも表示
- フッターに利用ソース一覧、VOICEVOXクレジット

### 生成中の表示
- ステップインジケータ:
  1. ニュース取得中(各ソースから取得した記事数を表示)
  2. 要約・台本作成中
  3. 音声生成中(N/Mセグメント)
- SSEで進捗をリアルタイム受信

### 再生画面
- 音声プレイヤー(再生/一時停止/シーク)
- 現在読み上げ中のニュース見出しを大きく表示
- 全ニュースリスト(タイトル、ソース、URL)

### `/history` (Phase 4)
- 日付ごとに過去の放送一覧
- 各エントリで再生ボタン

## 機能要件

### ニュース取得
- ソース定義は `docs/SOURCES.md` に集約
- 各ソースから過去24時間の記事を取得(タイムスタンプフィルタ)
- 重複記事の除去(URLとタイトルでデデュープ)
- 取得失敗ソースはスキップ、ログに記録

### 記事選定
- 取得した記事から**重要度上位3〜5件**をClaude APIで選ぶ
- 選定基準: 業界へのインパクト、独自性、読者(技術者)への有用性
- 同一トピックの複数記事は1件にまとめる

### 要約・台本化
- 各記事の本文(または要約)をClaudeに渡す
- アナウンサー風の自然な日本語台本に変換
- 全体で**約60秒(=日本語300〜400文字程度)**に収める
- 構成: オープニング(挨拶) → 各ニュース → クロージング
- 詳細プロンプトは `docs/PROMPTS.md`

### 音声合成
- 台本をセグメントに分割(オープニング・各ニュース・クロージング)
- VOICEVOX Engine HTTP APIで各セグメントを合成
- 単一の.wavファイルに結合(ffmpeg)
- セグメントごとの開始時刻をJSONで保持(再生中の見出し切り替えに使用)

### 再生
- HTMLの`<audio>`要素で単純再生
- `timeupdate`イベントで現在のセグメント特定 → 該当ニュース見出しをハイライト

## 非機能要件

- **生成時間**: ボタン押下から再生開始まで60秒以内が目標
- **コスト**: 1回の生成あたり想定$0.05〜0.15(Claude + VOICEVOXローカル)
- **オフライン**: VOICEVOXはローカルなのでネット不安定時もTTSは動く(ニュース取得・Claudeはネット必須)
- **MVPでは認証なし**、ローカル動作前提

## 制約事項

- **著作権配慮**: 各ニュースは要約に留め、原文転載しない。出典URL必須表示
- **VOICEVOXクレジット**: フッターに使用キャラクターの権利元表記
- **API利用規約**: 各ニュースソースのRSS/APIの利用規約に従う
- **公開時の追加対応**(Phase 5以降): レート制限、Anthropic API使用量上限
